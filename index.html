¿De que se trata?

Descripción del Proyecto

El objetivo de este proyecto es diseñar y construir una calculadora funcional utilizando assembly y un microcontrolador Arduino UNO. La calculadora debe ser capaz de tomar datos de un teclado de membrana 4x4 y realizar operaciones matemáticas básicas como suma, resta, multiplicación y división con estos para luego mostrar los resultados una pantalla LCD

¿Cúales son nuestros Objetivos?

Objetivos

Diseño

Diseñar el circuito de la calculadora en un simulador (tinkercad) para luego armarlo usando elementos físicos: Arduino UNO, pantalla LCD y teclado de membrana 4x4

Investigar

Investigar como implementar código de Assembly en Arduino

Assembly

Escribir el código en Assembly para recibir y procesar las entradas mediante el Arduino.

Implementación

Implementar las operaciones matemáticas básicas en el programa.

Metodologia

Primero, realizaremos en la aplicación tinkercad el diseño de las conexiones necesarias para el funcionamiento correcto de la calculadora tomando en cuenta los requerimientos de cada módulo. Una vez hecho esto, compraremos los materiales para armar el circuito y con la ayuda de un archivo (programa .ino) comprobaremos que este bien antes de codificarlo en assembly.

Después de esta prueba, programaremos en el lenguaje assembly todas las funciones necesarias para las operaciones aritméticas básicas como suma, resta, multiplicación y división; con este archivo listo, se escribirá un programa en el lenguaje C (lenguaje Arduino) que se encargue de invocar y controlar las funciones anteriormente descritas.

Finalmente realizaremos pruebas y ajustes necesarios para asegurar el correcto funcionamiento de la calculadora.

Nuestro Desarrollo

Desarrollo

Arduino UNO es una placa electrónica que funciona como una plataforma de prototipado para proyectos de electrónica y programación. Esta se encuentra equipada con un microcontrolador ATmega328P y puede programarse mediante un IDE basado en C++. Con una variedad de pines de entrada/salida y una interfaz USB, Arduino UNO es fácil de usar y permite a los usuarios conectar sensores, actuadores y otros dispositivos electrónicos para crear proyectos interactivos y automatizados. Para el presente proyecto los únicos módulos a utilizar serán un teclado matricial y una pantalla LCD.

Un teclado matricial 4x4 es un dispositivo de entrada que permite a los usuarios ingresar datos utilizando una matriz de 16 botones. Cada botón representa un valor numérico, alfanumérico o de control, y se conecta a la placa Arduino UNO a través de 8 pines.

Por otro lado, una pantalla LCD con módulo I2C es un dispositivo de salida que muestra información en texto y gráficos. El módulo I2C permite una comunicación serial de dos hilos entre la pantalla y la placa Arduino UNO.

Para una establecer una conexión efectiva y funcional se requiere la instalación de bibliotecas desde el IDE de Arduino: LiquidCrystal_I2C, Keyboard.

Problemas Encontrados

Las operaciones que se pueden realizar no pueden superar el entero 255; esto se debe a, como se mencionó antes, Arduino funciona a partir de un microprocesador ATmega328P, que es un procesador nativo de 8 bits, por lo tanto, no se puede superar el umbral mencionado trabajando directamente con Assembly.

Los registros en los que se guardan las entradas del usuario por teclado son muy específicos, al igual que los registros donde se almacenan los resultados con las operaciones. En este caso, se tuvo que buscar de manera minuciosa cuáles eran estos para acceder a su manejo.

El manejo de los números negativos fue algo complicado. Las salidas no eran las esperadas, por ejemplo, si operábamos -2 + 1, el resultado se disparaba a 255, por la misma razón que se definió anteriormente. Por esto, se tuvo que manejar este tipo de casos desde el programa C++ realizando algunas comprobaciones con condicionales.

Conclusiones

Arduino es una placa realmente conocida a nivel mundial debido a la facilidad con la que se pueden realizar proyectos de electrónica. Esta sencillez de desarrollo se debe al lenguaje con el que se trabaja, es decir, C++. Sin embargo, trabajar directamente con el microcontrolador puede llegar a ser una tarea ardua, cuya dificultad reside en investigar las propiedades de este.

Realizar este proyecto nos ha hecho saber las limitaciones que tiene la placa Arduino, a la vez, de lo bien optimizadas que se encuentran sus bibliotecas que manejan la enorme cantidad de módulos que existen, que van desde LEDS hasta kits de Machine Learning (tinyML, i.e.).
